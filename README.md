# Business Universe

Браузерная idle/tycoon-игра на React и TypeScript. Базовая игровая цепочка:

```text
деньги → картофельная ферма → картошка → завод чипсов → чипсы → деньги
```

Этот набор документации предназначен для **точечной разработки после завершения базового MVP**. В нём нет старой последовательности заданий для Cursor.

## Стек

- Vite;
- React;
- TypeScript в strict-режиме;
- Zustand;
- `localStorage`;
- CSS/CSS Modules;
- ESLint;
- Prettier.

## Запуск

```bash
npm install
npm run dev
```

## Проверки

```bash
npm run typecheck
npm run lint
npm run format:check
npm run build
```

Используй реальные команды из текущего `package.json`, если их названия отличаются.

## Документация

```text
docs/
├── PROJECT_STATE.md
├── BUSINESS_UNIVERSE_SPEC.md
├── ARCHITECTURE.md
├── DECISIONS.md
├── REGRESSION_CHECKLIST.md
├── BACKLOG.md
└── TASK_TEMPLATE.md
```

- `PROJECT_STATE.md` — что уже реализовано и какие есть ограничения.
- `BUSINESS_UNIVERSE_SPEC.md` — действующие игровые правила и числа.
- `ARCHITECTURE.md` — границы слоёв и устройство игрового цикла.
- `DECISIONS.md` — принятые архитектурные решения.
- `REGRESSION_CHECKLIST.md` — проверки существующего поведения.
- `BACKLOG.md` — будущие идеи, которые нельзя реализовывать без прямого запроса.
- `TASK_TEMPLATE.md` — шаблон для крупной точечной задачи.

## Правила работы с Cursor

Постоянные инструкции находятся в:

```text
.cursor/rules/business-universe.mdc
AGENTS.md
```

Cursor должен:

1. изучить текущую реализацию;
2. выполнить только явно запрошенное изменение;
3. сохранить постороннее поведение;
4. запустить релевантные проверки;
5. сообщить только реальные результаты;
6. остановиться после выполнения задачи.

## Сохранения

Текущая базовая схема использует ключ:

```text
business-universe-save
```

и версию:

```text
1
```

Любое изменение сохраняемых данных требует проверки обратной совместимости и, при необходимости, миграции.

## Как ставить крупную задачу

Для крупной функции можно временно создать:

```text
docs/CURRENT_TASK.md
```

на основе `docs/TASK_TEMPLATE.md`.

После завершения задачи перенеси важные решения в постоянную документацию и удали `CURRENT_TASK.md`, чтобы устаревшее задание не влияло на будущую работу.
